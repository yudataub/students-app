# מדור ניקוד התנהגות — עיצוב

תאריך: 2026-07-04

## מטרה

מדור חדש באפליקציית "זוכר שמות תלמידים" שמאפשר למורה לתת לכל תלמיד נקודות טובות (התנהגות חיובית) ונקודות רעות (התנהגות שלילית), כל אחת מתויגת באמוג'י ותיאור קצר, לספור אותן, ולמיין את התלמידים לפי שם או לפי ניקוד. פועל גם עבור תלמידים עם תמונה וגם בלי (משתמש ב-placeholder הקיים).

## מודל נתונים

### רשימת סוגי התנהגות (`behaviorTypes`)

מערך גלובלי חדש ב-localStorage, בפורמט:

```js
{ id: string, emoji: string, label: string, kind: 'good' | 'bad' }
```

ניתן לעריכה על ידי המורה (הוספה/מחיקה) דרך מסך ניהול. ברירת מחדל בטעינה ראשונה (אם המפתח לא קיים ב-localStorage):

**טובות:**
- ✅ הגיש שיעורי בית
- 🙋 השתתפות פעילה
- 🤝 עזר לחבר
- 📚 הביא ציוד לשיעור
- 🌟 התנהגות למופת

**רעות:**
- 🗣️ הפריע בשיעור
- 📵 טלפון בשיעור
- 😴 לא הקשיב / ישן
- ❌ לא הגיש שיעורי בית
- 🚫 חוסר כבוד

מחיקת סוג התנהגות מהרשימה אינה מוחקת רשומות עבר שכבר נוצרו ביומני התלמידים — היא רק מסירה את האפשרות מרשימת הבחירה להוספת נקודה חדשה.

### יומן נקודות לכל תלמיד (`student.behaviorLog`)

לכל אובייקט תלמיד קיים (במערך הגלובלי `students`) יתווסף שדה חדש:

```js
behaviorLog: [
  { id: string, typeId: string, emoji: string, label: string, kind: 'good'|'bad', ts: number }
]
```

- `emoji`/`label`/`kind` מועתקים מסוג ההתנהגות **בזמן ההוספה** (snapshot), כדי שהיסטוריה לא תשתנה גם אם המורה עורך/מוחק את הסוג מאוחר יותר.
- שדה חדש מאותחל כמערך ריק (`[]`) עבור תלמידים קיימים ב-migration בטעינה, באותו סגנון כמו המיגרציה הקיימת ל-`className`.

### חישוב ניקוד

הניקוד **לא נשמר כמונה נפרד** — הוא תמיד נגזר מהיומן:
- `goodCount(s) = s.behaviorLog.filter(e => e.kind === 'good').length`
- `badCount(s)  = s.behaviorLog.filter(e => e.kind === 'bad').length`
- `netScore(s)  = goodCount(s) - badCount(s)` (משמש למיון "לפי ניקוד")

## ניווט ומסכים

### כניסה למדור

- כפתור חדש `🏆 ניקוד` מתווסף לשורת הפעולות של כל כרטיס כיתה ב-`renderHome` (לצד גלריה/מבחן/הפוך/זיכרון/כיתה), באותו דפוס בדיוק כמו הכפתורים הקיימים (`onclick` קורא לפונקציה חדשה עם ה-`cls` המתאים).
- כפתור מקביל מתווסף גם לשורת "כל התלמידים" (`all-row`) בתחתית — פותח את המדור לכל התלמידים מכל הכיתות יחד, בדיוק כמו שכפתור הגלריה הכללי עובד היום.
- המדור **אינו** נגיש מכפתור עצמאי במסך הבית — רק דרך כרטיסי הכיתות/שורת "הכל", כמו הגלריה.

### מסך רשימה (`view-behavior`)

מסך חדש בדפוס דומה ל-`view-gallery`:
- כותרת: `🏆 ניקוד — <שם כיתה>` (או "כל התלמידים").
- כפתור `⚙️ ערוך רשימת התנהגויות` בראש המסך, פותח את חלונית הניהול (ראו למטה).
- שורת מיון עם שתי אפשרויות בלבד: **א-ב** (לפי שם פרטי, כמו המיון הקיים בגלריה) ו-**לפי ניקוד** (לפי `netScore` בסדר יורד, שוברי שוויון לפי א-ב).
- רשת כרטיסי תלמידים (שימוש חוזר ב-`gallery-grid`/`student-card` הקיימים), כל כרטיס מציג: תמונה (או placeholder הקיים לתלמידים בלי תמונה), שם, וסיכום ניקוד בשורה אחת: `✅ <goodCount>　🚩 <badCount>`.
- לחיצה על כרטיס פותחת את מסך הפרופיל של אותו תלמיד (ראו למטה).

### מסך פרופיל ניקוד לתלמיד

מסך חדש נפרד, `view-behavior-detail` (לא חלק ממסך הפרופיל הקיים `view-detail` של גלריה/מבחן — כדי לא לסבך את המסך הקיים בלוגיקה שלא קשורה אליו):
- תמונת התלמיד ושם, בראש.
- **סיכום קבוע למעלה:** `✅ <goodCount> נקודות טובות` ו-`🚩 <badCount> נקודות רעות`, תמיד גלוי.
- כפתור `➕ הוסף נקודה` — פותח בורר (overlay/רשימה) של כל סוגי ההתנהגות הפעילים (טובות ורעות יחד, כל אחת עם האמוג'י שלה), לחיצה על פריט מוסיפה רשומה ליומן התלמיד עם התאריך הנוכחי וסוגרת את הבורר מיד (ללא צעדים נוספים).
- כפתור `📋 הצג פירוט` — פותח/סוגר (toggle) תצוגת יומן מלאה: רשימת כל הרשומות, מהחדשה לישנה, כל שורה מציגה אמוג'י + תיאור + תאריך, עם כפתור `🗑️` למחיקת אותה רשומה בלבד (עם אישור `confirm()` פשוט לפני מחיקה, בדומה לפעולות מחיקה אחרות באפליקציה).

## ניהול רשימת ההתנהגויות (חלונית `⚙️ ערוך רשימת התנהגויות`)

Overlay בסגנון החלוניות הקיימות (`emoji-popup-overlay`, `settings-overlay`):
- שתי רשימות מוצגות: "נקודות טובות" ו"נקודות רעות", כל פריט עם אמוג'י + תיאור + כפתור `🗑️` למחיקה מהרשימה.
- טופס הוספה בתחתית: שדה טקסט לאמוג'י, שדה טקסט לתיאור, טוגל/רדיו לבחירת סוג (טובה/רעה), כפתור שמירה שמוסיף פריט חדש ל-`behaviorTypes` ושומר ל-localStorage.
- מחיקת פריט מהרשימה אינה נוגעת ברשומות עבר קיימות ביומנים (ראו מודל הנתונים למעלה).

## מחוץ לתחום (לא כלול בשלב זה)

- איפוס/reset תקופתי של הניקוד (שבועי/חודשי).
- ייצוא הניקוד לאקסל (ניתן להוסיף בעתיד באותו דפוס כמו ייצוא רשימת תלמידים הקיים).
- סנכרון ענן (Firebase) של `behaviorLog`/`behaviorTypes` — לא נדרש כרגע, בדומה לכך שהתכונה החדשה עובדת מקומית כמו שאר נתוני האפליקציה.
